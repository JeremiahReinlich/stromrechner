Erstelle eine mobile Web-App (Progressive Web App) zur Erfassung von Stromverbrauch mit Doppeltarif (HT/NT).

TECHNIK:
✓ Reines HTML, CSS, Vanilla JavaScript (kein Framework)
✓ Daten speichern im localStorage
✓ Mobile First Design
✓ Sticky Tabellenkopf
✓ Installierbar als PWA (manifest.json)

DATEIEN:
✓ index.html
✓ style.css
✓ app.js
✓ manifest.json

-------------------------------------

FUNKTIONEN:

1. TABELLE (Hauptansicht):
✓ Spalten: Datum | HT | NT | kWh | €
✓ HT und NT sind editierbar (Zählerstände)
✓ Alle anderen Felder sind berechnet (read-only)
✓ Tabelle zeigt jeden Tag (fortlaufend)
✓ Sticky Header beim Scrollen

-------------------------------------

2. LOGIK ZÄHLER & VERBRAUCH:

✓ Nutzer trägt HT/NT Zählerstände nur an manchen Tagen ein
✓ Leere Tage bleiben ohne Zählerstand
✓ Interpolation zwischen zwei Messpunkten
✓ Tagesverbrauch wird für ALLE Tage berechnet (inkl. Endtag)

Beispiel:
01.01 = 100
05.01 = 140
→ 02–05 bekommen jeweils 10 kWh

✓ HT/NT selbst werden NICHT interpoliert oder angezeigt
✓ Nur der Tagesverbrauch wird berechnet

-------------------------------------

✓ Beim Klick auf eine Zeile öffnet sich ein Detailbereich

Eingabefelder:

✓ Notiz (Text)

✓ HT/Tag
✓ NT/Tag

✓ HT Kosten

-------------------------------------

4. TARIFE (separater Bereich/Tab):

✓ Tarifstruktur:

  - HT Preis (€/kWh)
  - NT Preis (€/kWh)

✓ Regeln:
  - Tarife dürfen sich nicht überlappen
  - Pro Tag wird automatisch der passende Tarif verwendet

-------------------------------------

5. KOSTENBERECHNUNG:

✓ Pro Tag:
  - Grundpreis/Tag = Monatspreis / tatsächliche Tage im Monat
  - HT Kosten = HT/Tag * HT Preis
  - NT Kosten = NT/Tag * NT Preis
  - Gesamt = Grundpreis + HT + NT

✓ Grundpreis wird auch beim ersten Tag angezeigt (ohne vorherigen Wert)

13. ZUSÄTZLICHE FEATURES:

  - Undo stellt immer den vorherigen Zustand wieder her
  - Snapshot nur in der zentralen Daten-Update-Funktion erstellen (nicht im UI oder Render)
  - Undo darf keinen neuen Snapshot erzeugen

  - Speicherbedarf: ca. 300-500 KB für 10 Zustände (kein Problem für localStorage)
  - Desktop UI: Normaler Klick auf Undo-Button

  - Definition von "Änderung": Jede Operation, die den Datenzustand verändert

    * Löschen von Zusatzdaten
    * Speichern eines Tarifs (neuer Tarif)
    * Bearbeiten eines Tarifs
    * Löschen eines Tarifs
    * Importieren eines Backups (ersetzt alle Daten)
  - Keine Änderung (kein Snapshot): UI-Änderungen, Render-Operationen, Lese-Operationen

16. EINHEITLICHE BERECHNUNG (LÖSUNG):

Gewählte Lösung: Gemeinsame Berechnungsfunktion erstellen

Logik:
1. Wenn Werte eingetragen werden:
   - HT/Tag berechnen (aus Differenz zum vorherigen Wert)
   - NT/Tag berechnen (aus Differenz zum vorherigen Wert)
   - Grundpreis aus Tarif berechnen (tarif.grundpreis / tageImMonat)
   - HT Kosten berechnen (HT/Tag * tarif.htPreis)
   - NT Kosten berechnen (NT/Tag * tarif.ntPreis)
   - Kosten Gesamt berechnen (HT Kosten + NT Kosten + Grundpreis)

2. Wenn Werte fehlen:
   - HT/Tag interpolieren
   - NT/Tag interpolieren
   - Grundpreis aus Tarif berechnen (tarif.grundpreis / tageImMonat)
   - HT Kosten berechnen (HT/Tag * tarif.htPreis)
   - NT Kosten berechnen (NT/Tag * tarif.ntPreis)
   - Kosten Gesamt berechnen (HT Kosten + NT Kosten + Grundpreis)

Ohne Tarif: Kosten können nicht berechnet werden (zeigen "-" oder "Kein Tarif" an)

Umsetzung:
- Gemeinsame Berechnungsfunktion erstellen
- Tabelle verwendet diese Funktion
- Modal verwendet diese gleiche Funktion
- HT/Tag und NT/Tag werden interpoliert (wenn Werte fehlen)

Status: ✓ Erledigt
